# Foundation decisions - v0

## Что фиксируем прямо сейчас

### 1. Зависимости ядра
Ядро знает только интерфейсы:
- `ILLMProvider`
- `IMemoryService`
- `ITool`

Это ключевое архитектурное решение.

### 2. Memory на старте
На v0 память не включаем в full mode.

Причина:
- сначала нужен устойчивый основной pipeline
- mem0 добавим как следующий слой
- пока используем `NoOpMemoryService`, чтобы интерфейс уже был на месте

### 3. Planner на старте
Planner пока простой и детерминированный:
- определяет, нужны ли tools
- затем делает один вызов модели
- потом передает результат на верификацию

Это проще отлаживать, чем сразу строить сложный agent loop.

### 4. Tool execution
Tools исполняются через registry и dispatcher.

Это нужно, чтобы:
- легко добавлять новые инструменты
- централизованно логировать вызовы
- отделить планирование от исполнения

### 5. ClawGUI
ClawGUI не входит в ядро.

Он должен стать отдельным worker/adaptor позже.

### 6. State management
Session state пока in-memory.

Для v0 этого достаточно, потому что цель - проверить корректность архитектурного потока.
